## 🎮 게임 소개

**"Tetris VS"**는 테트리스를 플레이하는 **공격자(설계자)**와, 보드판 내부를 돌아다니며 생존하고 반격하는 **방어자(침입자)**가 대결하는 비대칭 액션 게임입니다. C 언어의 POSIX 소켓 통신과 멀티스레딩(pthreads) 기술을 활용해 구현되었습니다.

### 👥 플레이어 역할
* 🧱 **공격자 (Player 1):** 전통적인 테트리스 퍼즐을 플레이합니다. 위에서 아래로 블록을 떨어뜨려 방어자의 공간을 좁히고 깔아뭉개야 합니다. 
* 🏃 **방어자 (Player 2):** 테트리스 보드판 내부를 돌아다니는 액션 캐릭터(`@@`)를 플레이합니다. 보드가 가득 차지 않도록 블록을 치우면서, 아이템을 모아 천장에 있는 **공격자 보스**를 쓰러뜨려야 합니다.

---

## 🏆 승리 조건

* ☠️ **공격자의 승리 (탑아웃):** 보드판을 꼭대기까지 블록으로 꽉 채워서, 더 이상 블록이 내려오지 못하게 만들면 승리합니다.
* 🎯 **방어자의 승리 (보스 처치):** 맵에 등장하는 총(`G`) 아이템을 모아, 허공을 향해 **5발**을 발사하여 천장에 있는 공격자 보스의 체력(HP)을 0으로 만들면 승리합니다.

---

## 🚀 특수 아이템 시스템

떨어지는 테트리스 블록 중 **50% 확률**로 특수한 기호(`B`, `D`, `S`, `G`)가 박힌 아이템 블록이 등장합니다. 

### 🎒 3칸 인벤토리 & 채광 (Mining) 시스템
방어자는 최대 **3개**의 아이템을 보관할 수 있습니다. (FIFO 방식)
아이템은 블록을 직접 집어 들거나(`X`), **줄 파괴(Line Clear), 폭탄, 드릴**로 블록을 부수면 자동으로 획득됩니다!

| 아이템 | 기호 | 발동 효과 | 전략적 활용 |
|:---:|:---:|---|---|
| **폭탄** | `B` | 캐릭터 주변 4x4 크기의 블록을 순식간에 터트려 지워버립니다. | 갇혔을 때 최고의 탈출기! 파괴된 블록 속 아이템 흡수 가능. |
| **드릴** | `D` | 3초 동안 드릴 모드가 되어, 방향키로 블록을 뚫고 지나갈 수 있습니다. | 단단한 벽을 파괴하며 길을 개척하고 아이템을 채광합니다. |
| **방패** | `S` | 1초 동안 무적 쉴드(`[]`) 상태가 됩니다. | 이때 공격자가 블록으로 머리를 찍으면, **오히려 공격자가 1.5초 동안 기절**합니다! |
| **총** | `G` | 머리 위로 수직 상승하는 **총알(`^^`)**을 발사합니다. | 천장에 도달하면 보스 HP -1. 중간에 블록이 있으면 블록을 부숩니다. |

## 🕹️ 조작키 (Controls)

### 공격자 (Attacker / Tetris)
* `A` / `D` : 블록 좌우 이동
* `W` : 블록 회전
* `S` : 소프트 드롭 (조금 빨리 내리기)
* `Space` : 하드 드롭 (한 번에 바닥으로 내리기)
> **밸런스 룰:** 스페이스바(하드 드롭)를 누르거나 블록이 바닥에 고정되면, 다음 블록이 생성될 때까지 **0.6초의 딜레이**가 주어집니다.

### 방어자 (Defender / Character)
* `←` `↑` `→` `↓` : 캐릭터 이동 (상하좌우 화살표)
* `Z` : 점프 (물리 엔진 적용)
* `X` : 눈 앞의 블록을 집어 들거나 내려놓기 (Pick up / Place)
* `C` : 현재 소지한 첫 번째 아이템 사용하기

## 💻 빌드 및 실행 방법

> **권장 환경:** Ubuntu 24.04 (터미널 크기 **60x28** 이상 필수)
> **요구 사항:** GCC, `ncurses` 라이브러리, `pthreads`

```bash
# 1. ncurses 라이브러리 설치
sudo apt-get install libncurses5-dev

# 2. 게임 컴파일
make

# 3. 서버 실행 (터미널 창 1)
./server [port]   # 생략 시 기본 포트 9190 사용

# 4. 공격자 접속 (터미널 창 2)
./client 127.0.0.1

# 5. 방어자 접속 (터미널 창 3)
./client 127.0.0.1
```

## ⚙️ 적용된 시스템 콜 (System Calls)
- `socket()`, `bind()`, `listen()`, `accept()`, `connect()` - TCP/IP 소켓 통신
- `read()`, `write()` - 네트워크 패킷 송수신
- `open()`, `read()`, `write()`, `close()`, `stat()` - 로컬 하이스코어 파일 I/O
- `ioctl()` - 터미널 창 크기 실시간 확인
- `sigaction()` - 시그널(SIGINT, SIGTERM) 프로세스 핸들링
- `pthread_create()`, `pthread_mutex` - 멀티스레딩 (네트워크 수신 ↔ 렌더링/물리 엔진 스레드 분리)
